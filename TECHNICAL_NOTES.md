# 반려동물 모니터링 시스템 - 기술적 구현 참고사항

현재 데모 버전에서 **구현하지 않은** 기술적 요소들입니다. 실제 서비스 구축 시 아래 내용을 참고하여 개발이 필요합니다.

---

## 1. 실시간 영상 스트리밍

**현재 상태:** 플레이스홀더 영역만 표시

**실제 구현 시 필요 기술:**
- **WebRTC** 또는 **RTSP/HLS** 스트리밍 프로토콜
- 홈 카메라/디바이스에서 영상 전송하는 **미디어 서버** (예: Wowza, Ant Media, Janus)
- 또는 **IP 카메라** 직접 연동 (CORS, 보안 설정 필요)
- **백엔드 서버** (스트리밍 URL 관리, 인증)

---

## 2. 반려동물 감지 (AI/ML)

**현재 상태:** 10초마다 랜덤으로 상태 변경 (데모용)

**실제 구현 시 필요 기술:**
- **객체 감지 모델** (YOLO, TensorFlow Object Detection 등)
- **엣지 디바이스** 또는 **서버**에서 영상 분석
- 실시간 영상 프레임을 모델에 입력 → 반려동물 유무 판별
- **마지막 감지 시간**을 DB/서버에 저장 후 API로 제공

---

## 3. 음성 송출 (홈 기기로 전달)

**현재 상태:** 브라우저 Web Speech API로 사용자 기기에서만 TTS 재생

**실제 구현 시 필요 기술:**
- **홈 IoT 기기** (스피커/스마트홈 디바이스)
- **백엔드 API**: 텍스트 수신 → TTS 변환 또는 음성 파일 전달
- **실시간 통신**: WebSocket/MQTT로 홈 기기에 명령 전달
- 또는 **클라우드 TTS API** (Google Cloud TTS, AWS Polly 등)로 음성 생성 후 스트리밍

---

## 4. 사용자 인증 (보안)

**현재 상태:** 프론트엔드만, `sessionStorage` + 하드코딩된 데모 계정

**실제 구현 시 필요 기술:**
- **백엔드 인증 API** (JWT, 세션 기반)
- **비밀번호 해싱** (bcrypt 등)
- **HTTPS** 필수
- **로그인/로그아웃** 시 서버 세션 관리

---

## 5. 화면 캡처 (실제 영상 기준)

**현재 상태:** `html2canvas`로 현재 화면(플레이스홀더 포함) 캡처

**실제 스트리밍 연동 시:**
- `<video>` 요소에 실시간 스트림이 들어오면, `canvas.drawImage(video, ...)` 로 비디오 프레임 캡처 가능
- 단, **CORS** 이슈: 스트림이 다른 도메인이면 `crossOrigin` 설정 필요
- **보안**: 일부 브라우저는 외부 비디오 캡처 제한 가능

---

## 6. 메일 알림

**현재 상태:** 설정 UI만 구현 (On/Off, 수신 주소, 지연 시간) - localStorage에 저장

**실제 구현 시 필요 기술:**
- **백엔드 API**: 설정을 DB에 저장, 감지 서비스와 연동
- **메일 발송**: SMTP (Nodemailer, SendGrid 등) 또는 메일 API
- **감지 로직**: 반려동물이 N초 이상 미감지 시 메일 발송 트리거

---

## 7. 모터 추적 속도

**현재 상태:** 설정 UI만 구현 (느리게/중간/빠르게) - localStorage에 저장

**실제 구현 시 필요 기술:**
- **카메라/PTZ 모터** 하드웨어 연동
- **백엔드 API**: 모터 속도 파라미터를 디바이스에 전달
- **IoT 프로토콜**: MQTT, HTTP 등으로 제어 명령 전송
